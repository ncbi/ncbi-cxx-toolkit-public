Mac OS X (Darwin): Building the whole NCBI C++ tree locally

We have build NCBI C++ Toolkit using the prerelease GCC 3.1 compiler from the April 2002 Preview of Apple Developers Tools.

There are a few extra steps one have to follow before building the Toolkit on Mac OS X version 10.1. The future (10.2) version does not require these steps.

  • Download dlCompat library patch from (http://prdownloads.sourceforge.net/gnu-darwin). Copy the dlfcn.h include file into /usr/include directory. Copy libdl.dylib and libdl.a libraries into /usr/lib directory.
  • Select GCC 3.1 as a primary compiler (run "gcc_select 3").
  • Apply patch for C++ exceptions handling bug (Apple Radar bug #2466821). Replace crt1.o in /usr/lib with the latest version. The source code of crt1 (the Csu project) is avaliable from the Apple CVS. Please visit Apple Developer's page for information on how to obtaing Darwin source code from CVS. The indr utility is required for Csu build and is avaliable as a part of cctools project in Darwin CVS repository.

After these steps the procedure of configuring and building NCBI C++ Toolkit is the same as destribed in "Building NCBI C++ Toolkit: Running the configure script".
